今後の進めかた

これまでの講義/ワークをベースに、そろそろ本格的に自分のテーマでデジタルサービスの設計・制作に取り組みはじめます。コンセプト設計(AIと壁打ちしてもOK)から始まり、プロトタイプ制作〜検証、FigmaでのUI調整、そして動くアプリの実装まで段階的に進めましょう。

意味形成のデザイン ユーザー体験設計 Admin画面設計 社会的影響の考察 価値検証
使うツール（すべて無料）
Claude.ai

アイデア壁打ち・コンセプト設計・HTMLプロトタイプ生成。全員が全工程で使う。

Figma

ClaudeのHTMLをインポートしてUIを磨く。全員が第5回から使う。
プラグイン：html.to.design

Bolt.new Bプランのみ

FigmaデザインをもとにWebアプリとして実装・公開する。希望者のみ第6回から導入。

GitHub Pages Aプランのみ

完成したHTMLをURLとして公開する。Aプランの最終提出に使う。

第1段階　意味を掴む
Step1 テーマ探し
どんなものを作るのか考えていきましょう。これまでのワークで出たアイデアを練るのもよし、AIと壁打ち(トークン消費に注意)しながら考えるのもよし。この演習はアイデアが勝負ではなく、検証結果を含めて説得力を持たせることが重要です。アイデアはたくさん並行して検証してみましょーう！
Claude.ai
Step1のヒント

検証(プロトタイプテスト)のやり方も考慮しながらアイデアを選ぶ

→ 「読書習慣を身につけるためのサービス」なら「一定期間そのサービスを利用してみた後、習慣の変化をどのように記録するか」「サービス利用者と非利用者でどのように習慣化に差が出たか」「別々の機能でA/Bテストをした時にどちらが習慣化に寄与したか」なども含めてアイデアを練ってみましょう！

第2段階　サービスとして形にしていく
Step2 (デジタルサービスの場合は)UIをデザインする　Claude.ai → Figma
ある程度アイデアが固まり、(場合によってはペーパープロトタイプなどでも検証したら)Claude.aiでスマホアプリ風のHTMLを生成します。生成したHTMLをFigmaにインポートし（html.to.designプラグイン使用）、余白・フォント・色を自分の手で調整してみると良いでしょう。
Claude.ai Figma
📌 Figmaへの移行はhtml.to.designプラグインを使うと良いです！この辺もうまくAIと壁打ちしながら試行錯誤してみてください。
📌 このStep1〜2のアイディエーション、プロトタイプテストを繰り返しながら、「このサービスでいくぜ！」というものの当たりをつけていくのが当面の目標です。中間講評後にもそれなりに時間がありますし、AIも味方なので「このアイデアだけに絞るぜ！」と決め切らず、複数アイデアを並行して持っておくのが吉です。
次の次の回？ 中間講評
一旦どんな感じで進めているかを見せていただきます。発表ではなく、あくまでポートフォリオ的にまとめた資料"のみ"で提出してください。講評会でインタビューすることはあるかもしれません。

提出資料に盛り込む項目(コレ以外の項目を含んでもOK)

  • サービス概要
  • 制作プロセス
  • 現時点での検証内容・検証結果
  • 今後の方針
中間講評後に決める：デザインのみか、デザインも実装もしてみるか

どちらのプランでも最終成果物の評価基準は同じです。実装の有無ではなく、コンセプトの深さ・UIの質・社会的考察で評価します。が、検証は実装した方が精度が高いので、必然的に最終成果物のクオリティには寄与しそうです！

A Plan — デザインのみ
デザイン完成形を目指す
Claude.aiでHTMLを作り、Figmaで仕上げる。デザインの精度を高めることに集中する。最終成果物はFigmaデータ＋GitHub Pages公開のHTML(予定)
Claude.ai Figma GitHub Pages
B Plan — デザインと実装
動くアプリケーションとして実装する
Figmaのデザインを元にBolt.newで実装。ユーザー画面とAdmin画面を接続した動くアプリを作る。最終成果物はFigmaデータ＋Bolt.newのURL。
Claude.ai Figma Bolt.new
💡 途中でAプランからBプランに変更することもできます。ただし逆（B→A）は難しいので、迷ったらまずAプランで進めましょう。
